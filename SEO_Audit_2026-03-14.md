# Quansynd Website SEO Audit (Codebase Review)

Date: 2026-03-14  
Scope reviewed: `Quansynd-Web/` (Vite + React + React Router SPA)  
Production hosting (from Vercel CLI context): Vercel, custom domain `quansynd.com`

## Executive Summary

**Overall SEO score (out of 10): 5.8 / 10**

Strengths:
- Solid brand-level `<title>` + `<meta name="description">` on the homepage shell.
- Clean, crawlable URLs (React Router paths such as `/about`, `/services`, `/blog`).
- Content has meaningful headings (`<h1>`, `<h2>`) and generally descriptive image `alt` text in blog listings.

Primary blockers (highest impact):
1. **SPA rendering without prerender/SSR**: most routes do not ship unique HTML metadata to crawlers by default; everything is client-rendered.
2. **No per-route SEO metadata** (titles/descriptions/canonical/OpenGraph) for `/about`, `/services`, `/blog`, `/blog/:slug`, etc.
3. **No `sitemap.xml`** and `robots.txt` doesn’t reference a sitemap.
4. **No structured data (JSON-LD)** for Organization / WebSite / BlogPosting.
5. **Social sharing images are disabled** (OG/Twitter image tags commented out).

## Methodology (Best-Practice, Without Live Crawl)

Because this audit is based on the local repo (not a live crawl), it focuses on:
- Technical SEO: indexability, crawlability, routing, sitemaps, canonical strategy
- On-page: titles, descriptions, headings, internal linking, images
- Performance signals: bundle sizes, asset strategy, fonts/images
- Rich results + social: structured data, OG/Twitter tags

- GA4 analytics (traffic sources, landing pages, engagement)
- Backlink tool export (Ahrefs/Semrush) and competitor set

## Subagent Reviews (4 Perspectives)

### Subagent 1 — Technical SEO (Crawlability, Indexing, Rendering)

**Findings**
- `robots.txt` exists and allows crawling for all major bots, but **no sitemap directive**.
- **No `public/sitemap.xml`** (or generated sitemap) found.
- The site is a **React Router SPA**: initial HTML is a single shell (`index.html`) and most content/metadata is rendered client-side.
- Route-level `document.title` is only set in one place (QuanBench component), not consistently across pages.

**Risks**
- Google can index JS-rendered SPAs, but consistency is lower vs. SSR/SSG:
  - slower discovery/indexing of deep pages
  - weaker relevance scoring when `<title>`/description aren’t unique per page
  - poor social link previews when OG tags don’t match the shared URL

**Recommendations (Priority)**
- P0: Add a sitemap and reference it in `robots.txt`.
- P0: Add per-route titles/descriptions/canonical tags (at least).
- P1: Add prerender/SSG (preferred) or SSR if blog/service pages must compete on search.

---

### Subagent 2 — On-Page SEO (Content, Titles, Headings, Internal Links)

**Findings**
- Base `<title>` and description are good but global; they don’t describe specific pages like About/Services/Blog.
- Pages do use semantic headings:
  - `/` has an `<h1>` (“Designing Intelligence. Building the Future.”)
  - About/Services/Blog/Contact have `<h1>` with contextual copy.
- Blog content is embedded as HTML strings per slug (good for editorial control), but the **page metadata doesn’t reflect the post**.

**Recommendations**
- P0: Per-page `<title>` and `<meta name="description">`:
  - `/about`: “About QuanSynd — …”
  - `/services`: “AI Consulting & Design Services — …”
  - `/blog`: “QuanSynd Blog — …”
  - `/blog/:slug`: post title + excerpt
- P1: Add canonical URLs per route.
- P1: Ensure a single `<h1>` per page and consistent hierarchy.

---

### Subagent 3 — Performance & Core Web Vitals (SEO-Adjacent)

**Evidence (from recent Vercel build output)**
- `dist/assets/index-*.js` ~ **413 kB** (gzip ~ **125 kB**)
- `dist/assets/index-*.css` ~ **75 kB** (gzip ~ **14 kB**)
- Blog images range ~ **66–135 kB** (reasonable)

**Findings**
- External Google Fonts are loaded; this can affect LCP/CLS depending on font-display and caching.
- Many pages use large “hero” sections; LCP is often dominated by hero text/fonts/images.

**Recommendations**
- P1: Self-host fonts (or ensure `font-display: swap` + preloading critical fonts if needed).
- P1: Ensure hero images (if any) are optimized and not blocking LCP.
- P2: Route-split/lazy-load heavy sections where possible (especially blog post content and QuanBench).

---

### Subagent 4 — Rich Results, Social Previews, Measurement

**Findings**
- OpenGraph and Twitter tags exist, but **image tags are commented out**, so social sharing previews will be weak/inconsistent.
- No JSON-LD found for:
  - `Organization`
  - `WebSite` with `SearchAction` (optional)
  - `BlogPosting` for `/blog/:slug`
  - `BreadcrumbList` (optional)
- No obvious verification tags:
  - Google Search Console verification meta tag
  - Bing verification meta tag

**Recommendations**
- P0: Add OG/Twitter image tags with a real image hosted in `public/` (absolute URL on prod).
- P1: Add JSON-LD:
  - Organization schema site-wide
  - BlogPosting schema on blog post pages
- P1: Add Search Console verification (once you decide the GSC property).

## Prioritized Fix List (Action Plan)

### P0 (High impact, low-to-medium effort)
1. **Create `public/sitemap.xml`** (or generate at build-time) for:
   - `/`, `/about`, `/services`, `/blog`, `/contact`, `/quan-bench`
   - all blog slugs under `/blog/:slug`
2. Update `public/robots.txt` to include:
   - `Sitemap: https://quansynd.com/sitemap.xml`
3. Implement per-route SEO tags:
   - Title, description, canonical, OG/Twitter tags

### P1 (High impact, medium effort)
1. Add prerender/SSG for key routes:
   - at minimum, static routes + blog posts
2. Add JSON-LD for Organization + BlogPosting.
3. Add 301 redirect consistency rules (www ↔ non-www, trailing slash policy) if needed in Vercel.

### P2 (Incremental improvements)
1. Improve CWV (fonts, route splitting, image loading strategy).
2. Add RSS feed for blog (`/rss.xml`) (nice-to-have for discovery).
